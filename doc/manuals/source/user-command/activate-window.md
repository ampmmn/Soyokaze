# Switch Window Command

ウインドウをアクティブにしたり、位置やサイズを設定するためのコマンド。  
検索条件をあらかじめ設定しておき、それに対してキーワードやホットキーを設定することができる。  

このコマンドを使うと、以下の機能を実現できる。

- 特定のアプリのウインドウに切り替える
- コマンド実行時点で画面上に表示されているアプリのウインドウ一覧から選択し、切り替える

## Description

![](../image/edit-windowactivatecommand.png)

- `コマンドの名前`  
入力画面からコマンドを実行するためのキーワード
- `コマンドの説明`  
コメント表示欄に記載される文字列。

- `対象ウインドウの決定方法`  
コマンドを実行したときに操作を適用するウインドウの決定方法。  
`クラス名とキャプションから選択する` または `キーワード入力時にウインドウ一覧を列挙して選択する` のいずれかを選択する。
  - `クラス名とキャプションから選択する`  
ウインドウクラス名とタイトル(キャプション)を設定し、コマンド実行時に条件が合致するウインドウを探し、後述のアクションを適用する。  
右側にあるウインドウのアイコンを、対象とするウインドウへドラッグ&ドロップすることで、該当ウインドウのクラス名とタイトルを取得できる。
    - `ウインドウタイトル`  
検索対象とするウインドウのタイトルを指定する。  
空欄にした場合は`ウインドウクラス`のみで検索を行う。
    - `ウインドウクラス`  
検索対象とするウインドウクラスを指定する。  
空欄にした場合は`ウインドウタイトル`のみで検索を行う。
    - `ウインドウタイトル`と`ウインドウクラス`のどちらかを指定する必要がある
    - `正規表現`  
チェックした場合、`ウインドウタイトル`と`ウインドウクラス`に入力した文字列を正規表現として扱う。  
チェックしない場合、`ウインドウタイトル`と`ウインドウクラス`に入力した文字列の完全一致で探す。
    - `テスト`  
`ウインドウタイトル`、`ウインドウクラス`、`正規表現` に入力した内容に基づいてウインドウを探す。  
該当するウインドウが見つかった場合はタスクバー上にあるアイコンが点滅する。
  - `キーワード入力時にウインドウ一覧を列挙して選択する`  
コマンド名を入力したタイミングで、画面上に存在しているウインドウの一覧を列挙し、候補として表示する。  
選択した候補に対して後述のアクションを実行する。
- `対象ウインドウに対するアクション`  
選択したウインドウに対して適用する操作を設定する。  
右のアイコンをウインドウに対してドラッグ&ドロップすると、対象ウインドウの位置とサイズ情報を取得し、下記の入力欄に反映することができる。
  - `ウインドウをアクティブにする`  
ウインドウを前面に表示する。
  - `位置とサイズを変更する`  
選択したウインドウを、指定した位置・サイズに設定する。
    - `位置(X,Y)` `サイズ(幅,高さ)`  
画面上の位置、サイズを設定する。
    - `領域を確認する`  
指定した位置、サイズを示す枠を約2秒間ハイライト表示する。

- `ホットキー`  
コマンドを呼び出すキーバインドを設定できる。設定したキーを押下するとコマンドを実行できるようになる。
- `ホットキーからの実行専用にする`  
チェックした場合、キーワード入力でコマンド名がマッチしてもコマンドを候補欄に表示しない。  
(ホットキーを入力した場合のみ実行できるようにする)

#### Example

上の画面の設定でコマンドを作っておくと、下記の操作のいずれかで検索条件に該当するウインドウに切り替えることができる。  

- キーワード`actvim`を実行する
- (入力画面を表示した状態で)`Alt-V`押下

Windows標準のウインドウ切り替えのキー操作である`Alt`-`Tab`や`Win`-`Tab`の場合、以下の問題がある。

- その時々のウインドウの表示状態によりキー操作する回数が変わる
- ウインドウの数が増えると目的のウインドウを探すのに時間がかかる

このコマンドを使うと、画面にあるウインドウ数にかかわらず、定数時間で切り替えることができるメリットがある。


## Per-Key Behavior

|押下キー|動作|
|--|--|
|`Enter`|ウインドウを切り替える|
|`Ctrl-Enter`|ウインドウを切り替えて最大化する|
